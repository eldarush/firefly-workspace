# Adoption Rollout

## Week 0: Baseline

Interview senior architects and collect:

- last bad assistant moment;
- tasks they would never delegate;
- tasks they would gladly delegate;
- proof that would change their mind.

Baseline:

- active Claude Code users;
- sessions per user;
- accepted suggestions;
- PR/MR cycle time;
- Firefly workflow success rate;
- guardrail failures;
- drift and incident tickets;
- verification effort.

## Weeks 1-2: Architect Pilot

Give a small respected group narrow workflows:

- plan review;
- blast-radius review;
- drift investigation;
- guardrail authoring;
- codify unmanaged asset;
- CI componentization;
- SRE RCA;
- QA release packet.

The goal is one useful workflow, not generic prompt training.

## Weeks 3-4: Convert Prompts To Workflows

Use `/firefly-workspace:mine-skill` on successful sessions.

Every repeated prompt becomes one of:

- project context;
- approved memory;
- skill;
- agent;
- hook;
- eval;
- policy.

## Weeks 5-6: Team Rollout

Publish stable workflows through the internal marketplace. Run live clinics with real Firefly problems and before/after evidence.

## Weeks 7-8: ROI Readout

Report:

- adoption;
- first-pass usefulness;
- retries per task;
- verification coverage;
- unsafe-command blocks;
- accepted PR/MR outcomes;
- team sentiment;
- where AI saved time and where it moved work into review.
